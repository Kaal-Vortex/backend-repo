Task Assigned:
- Implement a response mapping layer to standardize API output
- Transform Tripjack raw JSON response into MakeMyTrip (MMT) room selection schema
- Ensure mapped response strictly follows MMT structure observed via Chrome DevTools

Understanding:
- Tripjack provides deeply nested raw data with pricing components (BF, SGP, TTSF, TMF, etc.)
- MMT expects a flattened, structured response focused on UI-friendly pricing fields (priceDetail)
- Pricing models differ:
  - Tripjack → component-based (base, tax, markup)
  - MMT → presentation-based (display price, discounted price, final price)
- Required to extract data from nested path:
  searchResult → his → ops → ris → pis → fc

Approach:
- First understand both schemas (Tripjack vs MMT)
- Identify correct mapping between pricing fields
- Build a reusable Python class for mapping
- Separate extraction logic and transformation logic
- Validate output against real MMT UI behavior
